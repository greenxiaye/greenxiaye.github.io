JS map 与 WeakMap

Map 和 weakMap 是 ES6 出现的数据结构。和 object 类似，都是可以键值对的存储

map

  1. 本质上是键值对的集合，类似集合
  2. 可以遍历，方法很多，可以跟各种数据格式转换

WeakMap

  1. 只接受对象作为键名（null 除外），不接受其他类型的值作为键名

  2. 键名指向的对象，不计入垃圾回收机制

    1. WeakMap 只能使用引用类型作为键，当这个键在其他地方没有被引用的时，这个键值对将会被回收

  3. 不能遍历，方法同 get、set、has、delete

    1. WeakMap 对象的键值对，任何时候可能被回收，所以没有内置迭代器。
    2. 注意，map_1[Symbol.iterator]和 map_1.entries 是一样的

总结 WeakMap 和 Map 的区别

  • 键的范围：Map 的键可以是任何类型的对象，支持设置为基本类型，而 WeakMap 的键只能是对象。
  • 键的保留：Map 的键会被保留，是强引用的对象，如果我们要拷贝的对象非常庞大时，使用 Map 会对内存造成非常大的额外消耗，而且我们需要手动清除 Map 的属性才能释放这块内存。WeakMap 的键则不同，如果没有其他引用指向它们，它们将被垃圾回收器自动回收，WeakMap 不会占用额外的内存空间。
  • 值的类型：Map 的值可以是任何类型的对象，而 WeakMap 的值只能是对象或 null。
  • 遍历方式：Map 可以被遍历，可以使用 for in 循环来遍历所有键和值。而 WeakMap 不能被遍历。

JS 深浅拷贝

原始数据类型和引用数据类型

js 的数据类型划分方式为 原始数据类型（基础数据类型）和 引用数据类型

  • 原始数据类型：Boolean、Number、String、Symbol、BigInt、Undefined、Null
  • 引用数据类型：Object、Array、Function、Date

原始数据类型和引用数据类型的区别

  1. 存储位置不同

    • 原始数据类型：存放在栈中，占据空间小、大小固定。
    • 引用数据类型：存放在堆中，因为占用空间大，大小不固定。

  2. 传值方式不同

    • 原始数据类型：按值传递。当使用一个原始类型的值时，实际上是在直接访问这个存储在栈中的值。

      • 不可变(immutable)

      • 方法操作无法改变一个基本类型的值

    • 引用数据类型：按引用传递。当使用一个引用类型的值时，引用数据会在栈中存放指向它的指针及内存地址，通过这个地址去堆中寻找对应的内容。

      • 引用类型的值是可变的

  3. 两种数据类型在实参和形参中的区别

    • 原始数据类型

    • 引用数据类型。在方法体内改变形参，将同时改变实参

浅拷贝与深拷贝

浅拷贝

  • 浅拷贝

    • 如果属性是基本类型，拷贝的就是基本类型的值。

    • 如果属性是引用类型，拷贝的是内存地址。这意味着，如果修改了引用数据的属性，那么浅拷贝的新对象也会受到影响，因为它们共享同一内存空间。

      • Object 浅拷贝

        1. Object.assign：用于对象的合并，将原对象(source)的所有可枚举属性，复制到目标对象(target)。
          如果源对象某个属性的值是对象，那么目标对象拷贝得到的是这个对象的引用.

        2. 拓展运算符：它和 Object.assign 类似

      • Array 浅拷贝

        1. Array.property.concat()

        2. Array.property.slice()

深拷贝

  • 深拷贝： 深拷贝是将一个对象从内存中完整的拷贝一份出来，从堆内存中开辟一个新的区域存放新对象，修改新对象不会影响原对象，因为新旧对象不共享同一内存空间。

  • JSON.parse(JSON.stringify())

    1. 其原理及为 JSON.stringify()将一个 JavaScript 对象转换为 JSON 字符串，使用 JSON.parse()将该 JSON 字符串解析为新的 JavaScript 对象，这个解析的过程会去开辟一个新的内容空间，即完成了深拷贝的过程。

    2. JSON.stringify()在转换的过程中，如果对象中包含函数、日期对象、正则表达式等无法被转换为字符串的类型，那么这些属性将被忽略或丢失，因而这些场景下是不适用的。

JS_EventLoop

JS_EventLoop

前提

  1. 宿主（node 或浏览器）发起的任务是宏观任务，JS 引擎发起的任务为微观任务。微观任务队列组成宏观任务
  2. JS 引擎常驻于内存，宿主把 JS 代码传给 JS 引擎去指向。宿主还会传 setTimeout 等 API 给 JS 引擎
  3. Promise 使得 JS 引擎自身发起任务，而不是宿主浏览器发起。
  4. 异步执行顺序，确定单个宏任务的微任务执行顺序，再确定宏任务的执行顺序，最后整个顺序

浏览器的 eventloop

（1）JS 的任务分为同步和异步。

  • 同步任务：在主线程上排队执行的任务，只有一个任务执行完毕，才能执行下一个任务。
  • 异步任务：不进入主线程，而是放在异步队列中，若有多个异步任务则需要在任务队列中排队等待，任务下一步会被移到执行栈，然后主线程执行调用栈的任务。
    • 执行栈：一个存储函数调用的栈结构，当我们执行 JS 代码的时候就是往执行栈中放入函数
    • 任务队列：数据结构中的队列结构，遵循先进先出的原则，主要负责将新的任务发送到队列中进行处理。
  • Tips
    在事件驱动的模式下，至少包含了一个执行循环来检测任务队列中是否有新任务。通过不断循环，去取出异步任务的回调来执行，这个过程就是事件循环，每一次循环就是一个事件周期。

（2） 任务队列根据任务种类的不同，可以分为宏任务与微任务。

  • 宏任务：scirpt 整体代码、setTimeout、setInterval、I/O、setImmediate

  • 微任务：Promise、MutaionObserver、process.nextTick

    本质区别：
    微任务：不需要特定的异步线程去执行，没有明确的异步任务去执行，只有回调
    宏任务：可能需要特定的异步线程（比如 setTimeout 就需要定时器线程）异步执行去执行

Tips

  1. async/await 函数产生 Promise，是微观任务
  2. async/await ，遇到 await 时就会退出执行，我想问下，退出之后是处于等待 await 执行完再开始之后
  3. 如果 promise 中产生 setTimeout 函数，那么在这里的 setTimeout 是宏观任务，因为你调用到了引擎以外的 API

node 的 eventloop

node 需要 Libuv

  • libuv 是一个跨平台异步 IO 库。
  • 因为 Nodejs 是单线程的，作为服务器，他涉及到 IO,而 IO 是会阻塞的，从而影响性能。所以 Nodejs 把 IO 操作交给 libuv，保证主线程可以继续处理其他事情。
  • Libuv 利用系统提供的事件驱动模块解决网络异步 IO，利用线程池解决文件 IO。另外还实现了定时器。
  • 定时器：本阶段执行已经被 setTimeout() 和 setInterval() 的调度回调函数。
  • 待定回调：执行延迟到下一个循环迭代的 I/O 回调。
  • idle, prepare：仅系统内部使用。
  • 轮询：检索新的 I/O 事件; 执行与 I/O 相关的回调（几乎所有情况下，除了关闭的回调函数，那些由计时器和 setImmediate() 调度的之外），其余情况 node 将在适当的时候在此阻塞。
  • 检测：setImmediate() 回调函数在这里执行。
  • 关闭的回调函数：一些关闭的回调函数，如：socket.on(‘close’, …)。

JS_map与forEach区别

map 与 forEach 区别

相同点

  1. 遍历数组
  2. 匿名函数中的 this 都是指 Window

不同点

  1. forEach 没有返回值，即返回值为 undefined

  2. map()返回一个新数组，原数组不会改变

    注意，如果 map 不写返回值，默认返回新的数组，每一项的元素是 undefined

  3. map 因为返回数组所以可以链式操作，forEach 不可以链式调用。

  4. map 速度比 forEach 快

箭头函数和普通函数

箭头函数的 this

  1. 箭头函数没有自己的 this，它的 this 是继承而来，默认指向在定义它时所处的对象(宿主对象)。注意是定义时的，不是执行时的对象。定义它时所处的对象是父级作用域，即父级普通函数的 this。由于箭头函数根本没有自己的 this，导致内部的 this 就是外层代码块的 this。

  3. 箭头函数继承来的 this 指向永远不会改变，call()、apply()、bind()等方法不能改变箭头函数中 this 的指向

箭头函数和普通函数区别

  1. 箭头函数中 this 的指向不同
    1.1 在普通函数中，this 总是指向调用它的对象，如果用作构造函数，它指向创建的对象实例
    1.2 箭头函数自身没有 this，它的 this 是继承而来，默认指向在定义它时所处的对象

  2. 箭头函数不能作为构造函数使用
    由于箭头函数时没有自己的 this，且 this 指向外层的执行环境，且不能改变指向，所以不能当做构造函数使用。

  3. 箭头函数没有自己的 arguments
    3.1 每一个普通函数调用后都具有一个 arguments 对象，用来存储实际传递的参数。
    3.2 箭头函数没有自己的 arguments 对象，取而代之用 rest 参数…解决。在箭头函数中访问 arguments 实际上获得的是它外层函数的 arguments 值。

  4. 箭头函数不具有 prototype 原型对象。
    箭头函数不具有 super，箭头函数不具有 new.target
    箭头函数不能 Generator 函数，不能使用 yeild 关键字。

  5. 箭头函数全都是匿名函数，普通函数可以有匿名函数，也可以有具名函数
