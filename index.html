JS_map、weakmap与object

JS map 与 WeakMap

Map 和 weakMap 是 ES6 出现的数据结构。和 object 类似，都是可以键值对的存储

map

  1. 本质上是键值对的集合，类似集合
  2. 可以遍历，方法很多，可以跟各种数据格式转换

WeakMap

  1. 只接受对象作为键名（null 除外），不接受其他类型的值作为键名

  2. 键名指向的对象，不计入垃圾回收机制

    1. WeakMap 只能使用引用类型作为键，当这个键在其他地方没有被引用的时，这个键值对将会被回收

  3. 不能遍历，方法同 get、set、has、delete

    1. WeakMap 对象的键值对，任何时候可能被回收，所以没有内置迭代器。
    2. 注意，map_1[Symbol.iterator]和 map_1.entries 是一样的

JS Map 与 Object

Map 与 Object 比较

  1. Object 只能使用字符串、数字、Symbol 作为 key，但 Map 可以任何数据类型作为 key

  2. Map 保证了插入顺序

    1. 代码示例
      迭代器里会有一个 next 函数, 可以 通过 next()去访问第一个。

  3. Map 内置了迭代器，可以 for-of

  4. Map 提供操作键值对的 api

  5. Map 使用类似 hash 表的算法实现了快速查找和删除操作，处理大量数据时比 object 高效

Map 与 Object 互转

  1. Object 转 Map

    1. 代码如下 new Map(Object.entries(obj))

  2. Map 转 Object

    1. 代码如下 Object.fromEntries(map.entries())

JS Set 与 WeakSet

Set

  1. 成员不能重复
  2. 只有键值，没有键名，有点类似数组
  3. 可以遍历，方法有 add、delete、has

WeakSet

  1. 成员都是对象（引用）
  2. 成员都是弱引用，随时可以消失（不计入垃圾回收机制）。可以用来保存 DOM 节点，不容易造成内存泄漏
  3. 不能遍历，方法哟欧 add、delete、has

JS 防抖与节流

防抖

触发高频事件后 n 秒内函数只会执行一次，如果 n 秒内高频事件再次被触发，则重新计算时间

节流

高频事件触发，但在 n 秒内只会执行一次，所以节流会稀释函数的执行效率

Tips

其实就是防抖靠定时器 timer 控制

JS闭包

JS 闭包

什么是闭包

  1. 简单理解，闭包只是一个绑定了执行环境的函数。
  2. 即一个函数和他周围状态（词法环境）的引用捆绑在一起的组合。
  3. 闭包可以让一个内层函数访问到其外层函数的作用域。（如在一个函数内创建另一个函数，创建的函数可以访问到当前函数的局部变量。）

闭包两个常用场景

  1. 创建私有变量

    1. 能在函数外部访问到函数内部的变量。
    2. 通过使用闭包，通过在外部调用闭包函数，从而在外部访问到函数内部的变量。这种方式来创建私有变量。

  2. 延长变量生命周期

    1. 一般函数的词法环境在函数返回后就被销毁，但是闭包会保存对创建时所在词法环境的引用，来延长变量的的生命周期。
    2. 因为闭包函数保留了这个变量对象的引用，使已经运行结束的函数上下文中的变量对象继续留在内存中，不会被回收。
    3. 下面的例子，treasure 函数执行完毕后，其执行环境生命周期会结束，所占内存会释放。但是通过 fn=innerTreasure，innerTreasure 的引用被保留下来，treasure 的变量 secret 也保留了下来。所以执行 treasureAgain 仍然能访问到 secret。
