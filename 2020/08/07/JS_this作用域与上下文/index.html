箭头函数和普通函数

箭头函数的 this

  1. 箭头函数没有自己的 this，它的 this 是继承而来，默认指向在定义它时所处的对象(宿主对象)。注意是定义时的，不是执行时的对象。定义它时所处的对象是父级作用域，即父级普通函数的 this。由于箭头函数根本没有自己的 this，导致内部的 this 就是外层代码块的 this。

  3. 箭头函数继承来的 this 指向永远不会改变，call()、apply()、bind()等方法不能改变箭头函数中 this 的指向

箭头函数和普通函数区别

  1. 箭头函数中 this 的指向不同
    1.1 在普通函数中，this 总是指向调用它的对象，如果用作构造函数，它指向创建的对象实例
    1.2 箭头函数自身没有 this，它的 this 是继承而来，默认指向在定义它时所处的对象

  2. 箭头函数不能作为构造函数使用
    由于箭头函数时没有自己的 this，且 this 指向外层的执行环境，且不能改变指向，所以不能当做构造函数使用。

  3. 箭头函数没有自己的 arguments
    3.1 每一个普通函数调用后都具有一个 arguments 对象，用来存储实际传递的参数。
    3.2 箭头函数没有自己的 arguments 对象，取而代之用 rest 参数…解决。在箭头函数中访问 arguments 实际上获得的是它外层函数的 arguments 值。

  4. 箭头函数不具有 prototype 原型对象。
    箭头函数不具有 super，箭头函数不具有 new.target
    箭头函数不能 Generator 函数，不能使用 yeild 关键字。

  5. 箭头函数全都是匿名函数，普通函数可以有匿名函数，也可以有具名函数
